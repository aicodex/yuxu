# yuxu Experiences — workflows + replay corpus

This folder is a **seed memory** for yuxu's iteration agent. Not auto-generated
— hand-curated from observed practice. Eventually the iteration agent consumes
it; today humans read it.

## Structure

```
experiences/
├── README.md                  # you are here
├── workflows/                 # reusable operational patterns discovered in practice
│   └── reference-driven-design.md
├── methodologies/             # distilled "how to do X" procedures
│   └── adding-architecture-principle.md
└── replays/                   # captured session units for regression testing
    ├── README.md              # replay corpus format
    └── 2026-04-24-memory-scoping-global-revive.md
```

## Why this exists

yuxu's architectural stance (I6, `docs/ARCHITECTURE.md`) is that memory is
primarily for the **iteration agent**: workflows + iteration outcomes, not a
user-facts database (OpenClaw) or coding-environment profile (Claude Code).
But the iteration agent doesn't exist yet. Meanwhile we are doing iteration
work by hand — debating architecture, rejecting my first proposals, cycling
through designs. That hand-work IS the experience the future agent needs to
learn from.

Two kinds of content:

- **Workflows / methodologies** — distilled patterns and procedures that
  should be reused. Future iteration agent reads these as guidance.
- **Replays** — captured raw units (context, user challenge, my response,
  what went suboptimally). Future iteration agent reads these as
  **test cases**: given the same context + challenge, can it do better?
  Especially valuable: the cases where I forgot earlier decisions and the
  user had to push back. Those are the highest-signal test units.

## Lifecycle

- Hand-curated today (2026-04-24).
- When memory infra Phase 1 lands (`memory.list` / `memory.get` /
  `memory.search`), each file here gets registered in the index and becomes
  queryable by the iteration agent.
- When the iteration agent ships, these files become regression benchmarks:
  given the context + challenge, iteration agent's response is LLM-judged
  against the "ideal" notes in each replay entry.

## What does NOT go here

- Active project memory (that goes in `<project>/data/memory/<agent>/`
  per I6)
- Agent-specific facts (same as above)
- Session transcripts (those go in `<project>/data/sessions/`, auto-
  generated by session_log)

This folder is the **teaching corpus**, not runtime memory.
